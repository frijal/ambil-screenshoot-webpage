Pengenalan SSH di Linux

SSH (Secure Shell) adalah protokol yang memungkinkan kita mengakses dan mengontrol komputer lain dari jarak jauh dengan aman. Biasanya digunakan untuk mengelola server Linux, mengirim file, atau menjalankan perintah jarak jauh.

Bayangkan kamu punya server di cloud. Dengan SSH, kamu cukup ketik perintah di terminal tanpa perlu GUI atau datang langsung ke lokasi server. Praktis dan cepat banget buat DevOps, SysAdmin, dan programmer.

SSH (Secure Shell) adalah salah satu alat paling penting di dunia Linux modern. Dengan SSH, kamu bisa mengakses komputer lain dari jarak jauh melalui jaringan dengan aman. Bayangkan kamu ingin masuk ke server di luar kota ‚Äî cukup buka terminal dan ketik perintah, tanpa harus datang ke lokasi.

SSH bukan cuma sekadar "login jarak jauh". Ia juga bisa digunakan untuk transfer file, menjalankan perintah otomatis, hingga mengatur keamanan antarserver. Jadi, bisa dibilang SSH itu adalah ‚Äúpintu rahasia‚Äù yang menghubungkan kamu dan server dengan cara yang terenkripsi.

Cara Kerja SSH

SSH bekerja dengan konsep **enkripsi** dan **autentikasi**. Saat kamu terhubung ke server menggunakan SSH, koneksi antara perangkat kamu dan server dilindungi oleh lapisan keamanan kriptografi. Semua data yang dikirim atau diterima tidak bisa dibaca oleh pihak lain.

Biasanya, koneksi SSH dibuat dengan perintah sederhana seperti:

ssh username@alamat-server

Setelah itu kamu akan diminta memasukkan password (atau bisa juga pakai kunci publik). Kalau berhasil, kamu langsung ‚Äúmasuk‚Äù ke server dan bisa menjalankan perintah seperti seolah-olah kamu ada di sana.

Membuat Kunci SSH (Public & Private Key)

Supaya login lebih cepat dan aman, banyak admin Linux menggunakan sistem kunci SSH. Caranya mudah: 

ssh-keygen -t rsa -b 4096

Setelah dijalankan, sistem akan membuat dua file:

  • id_rsa ‚Üí Kunci privat, simpan baik-baik di komputer kamu.
  • id_rsa.pub ‚Üí Kunci publik, bisa kamu unggah ke server tujuan.

Dengan begitu, kamu bisa login tanpa mengetik password lagi setiap kali. Selain itu, cara ini juga menghindari risiko password bocor atau brute-force.

Transfer File Menggunakan SSH

Selain untuk login, SSH juga bisa digunakan untuk menyalin file antar komputer dengan aman. Gunakan perintah: 

scp fileku.txt username@server:/home/username/

Kamu juga bisa melakukan backup otomatis atau sinkronisasi folder dengan perintah rsync yang berbasis SSH.

Keamanan SSH

Karena SSH membuka akses langsung ke server, maka keamanannya sangat penting. Beberapa hal yang perlu diperhatikan:

  • Gunakan port selain 22 agar tidak mudah ditebak.
  • Gunakan autentikasi berbasis kunci, bukan password.
  • Batasi akses pengguna dengan file /etc/ssh/sshd_config.
