Etika Kontribusi (Code of Conduct)

Proyek ini itu rumah kita bersama, jadi kita pengen lingkungannya tetap terbuka, ramah, dan inklusif buat semua orang. Intinya, kita semua di sini adalah teman-teman yang berniat baik!

  • Kita percaya semua orang yang ada di sini punya niat baik. Jadi, mari kita saling menghormati dan bersikap profesional ya.
  • Kalau kamu nemu atau ngelihat ada perilaku yang kurang pas, jangan ragu buat laporin via diskusi pribadi di GitHub. Privasimu aman kok!
  • Semua kontributor diharapkan banget buat baca dan patuhi CODE_OF_CONDUCT kita. Ini penting banget biar kita semua nyaman.

Bagaimana Cara Berkontribusi?

Eits, jangan salah! Berkontribusi itu gak melulu soal nulis kode, lho. Ada banyak cara seru lain yang bisa kamu lakuin buat bantuin proyek ini makin kece!

1. Pelaporan *Bug* (Masalah) üêõ

Pernah nemu error atau keanehan waktu browsing Layar Kosong? Atau ada link yang tiba-tiba putus? Nah, ini dia peranmu!

  • Gunakan tab Issues di GitHub. Ini tempatnya laporan bug!
  • Berikan deskripsi yang sejelas-jelasnya ya. Ceritain apa yang salah dan kalau bisa, kasih tahu langkah-langkah biar kita bisa nyoba ngulangin bug-nya.
  • Jangan lupa pakai label (bug) yang pas biar gampang dicari!

2. Diskusi & Ide Baru üí°

Punya ide gila buat fitur baru, gimana caranya bikin SEO makin ciamik, atau mau ngobrolin teknologi terkini? Jangan pendam sendiri!

  • Langsung aja ke tab Discussions di GitHub.
  • Ini adalah tempat paling pas buat *brainstorming* ide-ide brilian sebelum kamu mulai nulis satu baris kode pun. Siapa tahu ide dari kamu jadi fitur killer berikutnya!

3. Kontribusi Kode (Pull Request) üíª

Oke, buat kamu yang suka banget ngoprek kode, ini nih surganya! Baik itu perbaikan bug, update workflow, atau mempercantik CSS, semua diterima dengan tangan terbuka.

A. Persiapan

  1. Pertama, Fork Repository ini ke akun GitHub kamu. Anggap aja kamu nyalin buku buat diedit sendiri.
  2. Terus, Clone *repo* yang sudah kamu *fork* tadi ke komputer lokalmu. Biar bisa diedit offline.
  3. Buat branch baru buat perubahanmu (misalnya: feat/perbaikan-css atau fix/workflow-bug). Ini penting biar perubahanmu gak bentrok sama yang lain.

B. Aturan Penulisan Kode

  • HTML/Markdown: Kita suka yang simpel dan bersih. Pastikan kode HTML-mu valid dan mudah dibaca ya.
  • CSS: Karena ini proyek static site yang super ringan, usahakan pakai CSS yang efisien atau kalau perlu, pakai inline/scoped aja. Minimalis itu indah!
  • Workflow (.yml): Penting banget nih! Pastikan skrip YAML yang kamu bikin atau ubah itu berfungsi dengan baik dan pastinya aman ya.

C. Mengirimkan Pull Request (PR)

  • Pastikan commit kamu punya pesan yang jelas (misalnya: fix: Perbaiki link favicon yang rusak atau feat: Tambahkan action ping sitemap). Ini ngebantu banget waktu me-review.
  • Buka Pull Request ke branch main dari *repository* utama. Ini saatnya karyamu dilihat dunia!
  • Deskripsikan PR-mu dengan detail ya. Ceritain: Apa yang kamu ubah? Kenapa kamu mengubahnya? Dan yang paling penting, apa hasilnya?

Kontribusi Konten (Artikel)

Karena Layar Kosong punya fokus utama buat publikasi konten HTML yang kece, kami pastinya seneng banget kalau kamu mau ikutan nyumbang artikel atau tulisan. Yuk, bagikan ilmu atau cerita-ceritamu di sini!

  1. Bikin file HTML baru (contohnya: artikel-judul-mu.html).
  2. Taruh file tersebut di folder artikelx/.
  3. Kirimkan Pull Request seperti langkah di atas.

Penting Mengenai Lisensi Konten: Proyek ini memakai lisensi UNLICENSE (Public Domain). Itu artinya, dengan kamu mengirimkan artikel ke folder artikelx/, kamu secara otomatis melepaskan hak cipta atas tulisanmu itu ke Domain Publik. Pastikan kamu nyaman dengan keputusan ini ya!
