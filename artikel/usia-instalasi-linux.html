Script Sakti: Menguak Tanggal Lahir Partisi

Perhatian Penting!
Script ini bekerja di SEMUA Distro Linux, namun hanya bekerja jika partisi root kamu menggunakan filesystem ext2, ext3, atau ext4. Jika kamu pakai Btrfs, XFS, atau ZFS, gunakan metode alternatif di bagian bawah.

*Copy-paste* semua kode ini ke terminal kamu (jalankan sebagai *user* biasa). Script akan otomatis meminta *password* `sudo` untuk menjalankan `tune2fs`. 

#!/bin/bash

# 1. Menemukan device partisi root (e.g., /dev/sda2)
ROOT_DEV=$(findmnt -n -o SOURCE /)

# 2. Ekstrak tanggal pembuatan filesystem menggunakan tune2fs
# Menggunakan sudo karena tune2fs -l memerlukan izin root
# Bagian grep/cut/xargs mengambil string tanggal-waktu mentah
INSTALL_DATE_STRING=$(sudo tune2fs -l "$ROOT_DEV" 2>/dev/null | grep 'Filesystem created:' | cut -d: -f 2- | xargs)

# 3. Proses tanggal di Python dan hitung selisih hari
# (Catatan: tune2fs hanya bekerja andal untuk ext2/3/4)

python3 -c "
from datetime import datetime, date
import locale
# Atur locale ke 'C' untuk memastikan parsing tanggal konsisten (misalnya 'Mar' vs 'MÃ¤r')
try:
    locale.setlocale(locale.LC_ALL, 'C')

    # Parsing format tanggal seperti 'Sat Mar 16 14:30:22 2024'
    creation_datetime = datetime.strptime('$INSTALL_DATE_STRING', '%a %b %d %H:%M:%S %Y')
    d1 = creation_datetime.date()
    d2 = date.today()
    print(f'\n Tanggal Instalasi Filesystem: {d1}')
    print(f' Usia Sistem Anda adalah {(d2-d1).days} hari.\n')
except ValueError:
    print(' ERROR: Tidak dapat mengambil tanggal pembuatan filesystem dari tune2fs.')
    print('Hal ini terjadi jika filesystem Anda tidak menyimpan creation time (misalnya XFS atau Btrfs).')
    print('Silakan gunakan metode alternatif (Log Paket) di bagian bawah.\n')
"

Cara Script Ini Bekerja (Detail Buat Para Suhu)

Di balik kesederhanaan *copy-paste*, ada beberapa *tool* terminal yang bekerja keras:

  1. **`findmnt -n -o SOURCE /`**: Script ini fleksibel karena ia tidak berasumsi nama partisi. Ia mencari tahu di *device* mana (`SOURCE`) *root* (`/`) kamu terpasang.
  2. **`sudo tune2fs -l "$ROOT_DEV"`**: Ini adalah kunci utamanya. `tune2fs -l` (list superblock) mampu membaca metadata historis dari *filesystem* keluarga ext, termasuk item yang sangat berguna: `Filesystem created:`.
  3. **Pembersihan Output (`grep`, `cut`, `xargs`)**: Bagian Bash ini bertugas membersihkan output `tune2fs` menjadi string tanggal mentah yang siap disantap oleh Python.
  4. **Python `datetime`**: Python digunakan untuk memproses string tanggal, membandingkannya dengan tanggal hari ini (`date.today()`), dan menghitung selisihnya dalam satuan hari. Hasilnya pasti akurat!

Alternatif untuk Filesystem Non-Ext (Cek Log Paket)

Tips: Kalau kamu pakai Btrfs, XFS, atau ZFS, atau jika script di atas gagal, kamu masih bisa mendapatkan perkiraan usia instalasi kamu dengan mencari tanggal instalasi paket sistem dasar yang paling awal di log package manager kamu.

Untuk Arch/Manjaro (Pacman):

Cari *timestamp* instalasi paket `filesystem` atau `linux` yang paling awal di log Pacman:

grep -E '\[INSTALL\] filesystem|\[INSTALL\] linux' /var/log/pacman.log | head -n 1

Untuk Debian/Ubuntu/Mint (APT):

Cek log instalasi APT yang mencatat paket `base-files` atau `linux-image`:

grep ' install ' /var/log/dpkg.log | head -n 1

Contoh outputnya akan menunjukkan tanggal dan waktu yang sangat dekat dengan momen instalasi Linux kamu:

2024-03-16 14:30:22 install base-files:amd64 12.0

Tanggal di awal baris itulah yang bisa kamu jadikan patokan! Sekarang tinggal hitung deh usianya.
